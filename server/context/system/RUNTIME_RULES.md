# RUNTIME_RULES.md

## Runtime 目标

每个已发布 app 应在共享平台依赖上稳定运行，同时保持 app 代码与数据隔离。

## 规则

- 平台依赖可共享（node_modules、基础镜像、宿主框架）
- app 代码与数据库必须独立存放于 `server/apps/<id>/`
- runtime 健康不应只看进程存在，还应检查业务接口可用性
- app backend 重启后不应依赖偶然残留状态才能工作
- schema apply 必须可幂等执行

## 兼容性

- 老版本字段迁移应优先通过兼容迁移解决
- 兼容补丁应优先进发布流程，而不是事后手工修库
- 历史 app 如依赖公共运行时，升级平台依赖时必须考虑回归风险
